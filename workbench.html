Designed and built from scratch

This portfolio website

A lightweight, responsive static site built with plain HTML, a shared CSS design system and a tiny amount of JavaScript.

HTML · CSS · JavaScript · Responsive design

Simple on purpose, not simple by accident.

I designed and built this site to present my work clearly without hiding it inside a large framework or complicated build system. Each page is ordinary semantic HTML, the visual language lives in one shared stylesheet and JavaScript is used only where it adds something useful.

The navigation is the main shared JavaScript component. Everything else is handled by the browser using HTML and CSS: layout, spacing, typography, responsive behaviour, project cards and visual hierarchy.

I chose this approach because the site does not need application state, a database or client-side rendering. A static portfolio should load quickly, remain easy to understand and be simple to change without introducing unnecessary dependencies.

Why build it this way?

The architecture should match the size of the problem.

A portfolio is mostly structured content. It needs strong visual design, clear navigation and responsive layouts, but it does not need the machinery of a full web application.

Keeping the implementation small makes the site easier to debug, faster to load and much harder to break. It also means I can open any page and immediately see how it works.

The simplicity is also part of the design. The pages share the same colours, typography, spacing, cards and layout patterns, so new content can be added by reusing an established system rather than inventing each page again.

Junction Playground

Junction Playground

A playful exploration of Junction’s normalised health API. Using sandbox sleep data from an Oura Ring, Fitbit Watch, Fitbit App and Apple Health, the demo compares how four sources interpret the same person’s sleep. It calculates nightly averages, highlights gaps and disagreements, and includes a conversational AI agent that analyses the most reliable dataset and answers questions about sleep patterns.

Built with Next.js, TypeScript, the Junction SDK and the OpenAI Agents SDK.

  • API inspection
  • Human-readable findings
  • A little Chatbot to talk through the results with
